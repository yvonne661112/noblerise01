# format-article — Markdown 文章排版規範

對 `src/content/blog/` 下的文章進行排版整理時，依照以下規則執行。

Arguments: `$ARGUMENTS`（可指定單一檔案路徑，或不填則處理全部）

---

## 規則一：Frontmatter 完整性

每篇文章必須包含以下四個欄位，缺漏者補齊：

```yaml
---
title: "文章標題"
description: "摘要（100字以內）"
date: YYYY-MM-DD
tags:
  - "標籤一"
  - "標籤二"
---
```

其他欄位（`numericSlug`、`image`、`categories`）若已存在則保留，不得刪除。

---

## 規則二：標題層級

- h2（`##`）作為主要段落標題
- h3（`###`）作為子項標題
- **不使用 h4（`####`）以下層級**，若有則升為 h3 或合併至段落
- **不使用 h1（`#`）**，文章標題已在 frontmatter `title` 欄位

---

## 規則三：清單

- 有序清單（`1.`）用於步驟流程（有先後順序）
- 無序清單（`-`）用於並列項目（無先後順序）

---

## 規則四：強調與引言

- 重要名詞、金額、專有詞彙用 `**粗體**`
- 警示、引言、名言用 `>` 區塊引言

---

## 規則五：表格

數字比較類內容（如費率對照、商品比較）改為 Markdown 表格格式：

```markdown
| 項目 | 數值 |
|------|------|
| 保費 | 15萬 |
```

---

## 規則六：空行與斷行

1. 每個 h2 段落**前**空一行
2. 內文段落**間**空一行
3. 連續兩個以上的空白行，合併為一個空白行
4. 同一語意未結束即斷行者（句子中間的不合理換行），合併回同一段落

---

## 規則七：禁止偽標題

不可用獨立一行的 `**粗體**` 代替標題，應：
- 升為對應的 `##` 或 `###`，**或**
- 合併為段落開頭粗體（接續後文）

**錯誤範例：**
```
**保險規劃重點**

說明文字...
```

**正確範例（升為標題）：**
```
## 保險規劃重點

說明文字...
```

**正確範例（合併為段落）：**
```
**保險規劃重點**：說明文字...
```

---

## 規則八：不修改中文文字內容

僅調整排版格式，**不得改動任何正文文字**，包含：
- 不改字、不加字、不刪字
- 不調整語句順序
- 不翻譯或改寫

---

## 執行方式

### 單一檔案
```
/format-article src/content/blog/20260328.md
```

### 全部檔案（批次）
```
/format-article
```
批次處理時，先掃描所有檔案找出不符規則之處，再依批次平行處理，完成後執行 `npm run build` 驗證。

---

- [ ] 無 h4 以下標題
- [ ] 無獨立一行的偽標題粗體
- [ ] 空行符合規範
- [ ] 中文文字內容未被修改
